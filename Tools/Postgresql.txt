By default, PostgreSQL uses an authentication method called peer which might not work easily with Django because Django connects to PostgreSQL using database username, password over a TCP/IP connection (localhost or 127.0.0.1). This requires password-based authentication, not peer. So if your PostgreSQL is still using peer for local connections, Django will fail to connect with errors like: FATAL:  Peer authentication failed for user "uloit_admin"

Open the config file: vi /var/lib/pgsql/data/pg_hba.conf
Change peer and ident to scram-sha-256: 

# "local" is for Unix domain socket connections only
local   all             all                                     scram-sha-256
# IPv4 local connections:
host    all             all             127.0.0.1/32            scram-sha-256
# IPv6 local connections:
host    all             all             ::1/128                 scram-sha-256
